# 更新说明

本项目的显著变更记录。版本号对应 `project.yml` 的 `MARKETING_VERSION`。

## 0.1（开发中）

### 2026-07-03

**修复**

- **息屏 Live Activity 不跟随提前切换**：提前结束进食（或提前开始断食）后，锁屏仍在倒数旧窗口的时间。两个成因一起修：(1) 重装 app 或 iOS 的 8 小时上限会在活动列表里留下「僵尸」activity——对已结束的实例调用 update 是静默无效的，而它占着列表挡住了「重新新建」的判断；同步逻辑改为按存活状态分流：更新第一个活着的、清扫其余全部、一个活的都没有才重新新建。(2) iOS 只允许 app 在**前台**新建 Live Activity，后台启动时的那次同步失败后原逻辑不会重试；现在每次回到前台都强制重新同步一次，任何一次丢失的同步都会在下次打开 app 时自愈。另为 DEBUG 构建加了同步诊断记录（时间戳 + updated/requested/失败原因）。

### 2026-07-02

**新增**

- **自噬能量球 · 蚂蚁森林式超时奖励**：断食超过目标后，每 30 分钟在松鼠周围长出一颗可点击收集的金色 +1 能量球（同屏最多 8 颗，超出排队补位）；结束断食时未收集的球自动结算入账，不惩罚忘点。主界面新增「自噬能量」卡实时显示余额；余额攒到阈值弹「能量满啦」，可手动兑换一顿放纵餐（兑换扣阈值点数，余下继续攒）。阈值在设置新增的「放纵餐」节调整（5–50 点、步进 5、默认 10）。球数为纯派生值（超时时长 ÷ 30min − 已收集），无定时器、重启 app 分毫不差。新增调试启动参数 `-seedOvertime` 直达超时状态。

**修复**

- **息屏 / 灵动岛 / 小组件图标随相位切换**：此前 5 处硬编码 `moon.zzz.fill`，无论断食还是进食都显示睡月亮。现在断食显示月亮 `moon.zzz.fill`（橙）、进食显示刀叉 `fork.knife`（绿），图标定义收敛到 `Phase.symbol` 一处，后续换图标只改一行。
- **进度环改为「时间诚实」**：此前五个阶段节点均匀 72° 等分，而阶段真实时长悬殊（16:8 下「饱足」仅 1.9h、「消化」6.1h），弧速忽快忽慢最多相差 5 倍，弧的位置与实际时间不成比例。现在节点按各阶段真实开始时刻摆放（角度 = 时刻 ÷ 24h × 360°），弧匀速行进、弧长与时间严格成正比，且弧扫到节点的瞬间仍正好是阶段切换时刻。节点位置随节律（16:8 / 18:6 / 20:4）自适应；相邻过近的标签（如短阶段「苏醒」）自动外推防重叠。

**行为说明（文档化既有设计）**

- 息屏 / 小组件的断食/进食判定基于**你实际点按的时刻**：有进行中的会话即为断食，否则锚定上次结束时刻 + 进食时长；设置中的节律只决定目标时长，不绑定钟点。窗口到点后**不自动翻相位**，始终由你手动点按切换（有意为之）。

### 2026-06-29 及更早

- 蚂蚁森林式功能上线前的基础版本：会代谢的松鼠吉祥物（5 阶段姿势/呼吸/粒子）、经典与矢量双皮肤、一键断食/进食切换（循环锚定点按时刻）、16:8 / 18:6 / 20:4 节律、统计面板、锁屏与灵动岛 Live Activity、控制中心 / Siri / Action Button 快捷切换、四尺寸桌面小组件、本地通知；全部数据本地存储。
